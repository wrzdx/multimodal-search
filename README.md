# Multimodal Search Alliance for Trading Strategies

Course project for **Deep Learning for Search (Summer 2026)**. The system retrieves trading strategies using both natural-language descriptions and equity curves, combines sparse and dense retrieval with Reciprocal Rank Fusion, and reranks candidates with a LightGBM LambdaMART model ("The Captain"). A Streamlit interface provides search, risk filters, visualized curves, and a lightweight faithfulness check.

## Architecture

1. **Data collection and cleaning** — `real_parser.py` and `data_parser.py` build a real-world corpus and normalize it into Parquet.
2. **Sparse retrieval** — BM25 is always available; SPLADE is optional.
3. **Dense retrieval** — a text bi-encoder and curve encoder produce FAISS indices.
4. **Fusion and reranking** — RRF combines candidates and LambdaMART reranks them using text, curve, and financial features.
5. **Feedback and UI** — Agentic CRAG-style filtering refines results from user constraints; Streamlit displays the final ranking.
6. **Evaluation** — `evaluate.py` compares retrieval variants and logs ranking quality, latency, and hardware usage.

## Repository layout

```text
multimodal-search/   Python retrieval, training, evaluation, and UI code
```

Datasets, embeddings, caches, and FAISS/BM25 indices are intentionally not stored in Git. They are large, reproducible artifacts and can exceed GitHub's file-size limit.

## Installation

Python 3.10–3.12 is recommended.

```bash
cd multimodal-search
python -m venv .venv
```

Activate the environment:

```powershell
# Windows PowerShell
.\.venv\Scripts\Activate.ps1
```

```bash
# Linux/macOS
source .venv/bin/activate
```

Install dependencies:

```bash
python -m pip install --upgrade pip
pip install -r requirements.txt
```

## Reproducing data and indices

Run all commands from the `multimodal-search` directory.

### 1. Collect at least 500,000 real documents

The fastest default source is Hugging Face datasets:

```bash
python real_parser.py --sources hf,arxiv,openalex --target 500000
python data_parser.py --skip-generate
```

This creates `raw_strategies.jsonl` and `clean_strategies.parquet`. Network availability and upstream rate limits affect collection time. Do not use `--skip-generate` unless `raw_strategies.jsonl` already exists.

### 2. Build the sparse index
Because we do not have token for SPLADE
```bash
python sparse_index.py --skip-splade
```

BM25 works without gated models. To enable SPLADE, obtain access to the model named in `sparse_index.py`, authenticate with Hugging Face, and run the command without `--skip-splade`.

### 3. Train the encoders

```bash
python train.py
```

For a shorter diagnostic run:

```bash
python train.py --epochs 3
```

Training produces `curve_encoder.pth` and `text_proj_head.pth`. The full run can take hours on CPU.

### 4. Build dense FAISS indices

```bash
python dense_index.py
```

The comparative study writes metrics and hardware profiling results to `experiments_log.jsonl`.

### 5. Launch the interface

```bash
streamlit run app.py
```

## Generated files not committed to Git

The following files remain on the developer's machine and are regenerated by the pipeline:

- `raw_strategies*.jsonl`
- `clean_strategies.parquet`
- `bm25_index.pkl`
- `faiss_*.index`
- `*_embeddings.npy`
- `hard_negatives_cache.npy`
- optional SPLADE matrix/vocabulary files

Small trained model files (`curve_encoder.pth`, `text_proj_head.pth`, and `captain_lgbm.txt`) are included so the project configuration and lightweight learned artifacts remain reproducible and reviewable.

## Hardware

- **Inference:** CPU-only operation is supported; 16 GB RAM is recommended for a 50k+ corpus.
- **Development/training:** an NVIDIA GPU with CUDA and at least 8 GB VRAM is recommended, but CPU training is supported and substantially slower.
- Storage requirements depend on corpus size; allow several gigabytes for the dataset, embeddings, model caches, and indices.
